# Superhuman Labels & Inbox Shaping Endpoints

Last updated: 2026-03-23

## Scope

Endpoints for label sync, auto-archive rules, AI auto-labels, importance overrides, bulk relabeling, split inboxes, and synthetic inboxes.

---

## Label sync

### `labels.resync`

Force a full re-sync of labels from the provider (Gmail/Outlook).

| Field | Value |
|-------|-------|
| **URL** | `POST /~backend/v3/labels.resync` |
| **Method** | POST |
| **Body** | None |
| **Retries** | Up to 4 attempts |

### `labels.recentChanges`

Get recently changed labels (Microsoft/Outlook only).

| Field | Value |
|-------|-------|
| **URL** | `GET /~backend/v3/labels.recentChanges?historyToken=<token>` |
| **Method** | GET |
| **Retries** | Up to 4 attempts |

Returns `410` with `expired-history-token` if the token is stale.

## AI auto-labels

### `autoLabels.upsert`

Create or update an AI auto-label rule.

| Field | Value |
|-------|-------|
| **URL** | `POST /~backend/v3/autoLabels.upsert` |

```json
{
  "id": "<label_id>",
  "name": "Invoices",
  "prompt": "Emails containing invoices or payment requests",
  "deterministicCriteriaGroups": [...],
  "summary": "<auto-generated summary>"
}
```

The `deterministicCriteriaGroups` field is processed by `_processDeterministicCriteriaGroups()` before sending. The `summary` is auto-generated from the rule definition.

### `autoLabels.delete`

Delete an auto-label rule by ID.

| Field | Value |
|-------|-------|
| **URL** | `POST /~backend/v3/autoLabels.delete` |

```json
{ "id": "<label_id>" }
```

### `autoLabels.modifyConfig`

Update auto-label configuration.

| Field | Value |
|-------|-------|
| **URL** | `POST /~backend/v3/autoLabels.modifyConfig` |

Body is a config object (shape not fully visible). Accepts a `traceId` for metrics.

### `autoLabels.preview`

Preview which emails an auto-label rule would match. **Streaming response.**

| Field | Value |
|-------|-------|
| **URL** | `POST /~backend/v3/autoLabels.preview` |
| **Response** | **Streaming** |

```json
{
  "id": "<label_id>",
  "name": "Invoices",
  "prompt": "Emails containing invoices",
  "deterministicCriteriaGroups": [...]
}
```

Supports `AbortController` signal.

## Bulk relabeling

### `relabels.create`

Create a bulk relabeling action (move emails matching a query).

| Field | Value |
|-------|-------|
| **URL** | `POST /~backend/v3/relabels.create` |

```json
{
  "id": "<relabel_id>",
  "query": "from:newsletters@example.com",
  "addLabelIds": ["Label_123"],
  "removeLabelIds": ["INBOX"],
  "actionType": "archive" | "move" | "label",
  "includeThreadIds": [],
  "excludeThreadIds": [],
  "splits": [...],
  "labels": [...],
  "maxThreadTimestamp": "..."
}
```

Returns `{ "relabelId": "<id>" }`.

### `relabels.revert`

Undo a bulk relabeling action.

| Field | Value |
|-------|-------|
| **URL** | `POST /~backend/v3/relabels.revert` |

```json
{ "relabelId": "<id>" }
```

Returns 404 if the relabel has already expired.

| Field | Value |
|-------|-------|
| **URL** | `POST /~backend/v3/syntheticInbox.reset` |
| **Body** | None |
